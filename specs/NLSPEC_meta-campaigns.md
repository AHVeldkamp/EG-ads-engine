## 1. Context

### 1.1 Problem Statement

enduro-gear.nl needs to create and manage Meta (Facebook/Instagram) advertising
campaigns programmatically. The system must manage the full Meta ads hierarchy
(Campaign > Ad Set > Ad), store all data locally in PostgreSQL, and push each
level to Meta's Marketing API via dedicated publish endpoints. Image creatives
generated by the existing `creative-generation` module must be uploaded to Meta
when publishing ads.

### 1.2 Prior Art

- **creative-generation module** — Generates ad images via Google Gemini and
  stores them at `uploads/creatives/{creativeId}.png`. The `Ad` model will
  reference `Creative` via a foreign key. The creative-generation module
  establishes the codebase patterns for controllers, services, DTOs, pagination,
  and Prisma usage that this module must follow.
- **Meta Marketing API** — Facebook's API for programmatic ad management.
  Requires an access token, app credentials, ad account ID, and page ID.

### 1.3 Scope

**IN scope:**
- Full CRUD for Campaigns, Ad Sets, and Ads (local PostgreSQL)
- Per-level publish endpoints to push each entity to Meta's Marketing API
- Image upload to Meta (from local creative files) when publishing an Ad
- Meta creative creation (linking uploaded image to ad copy)
- Track Meta IDs (metaCampaignId, metaAdSetId, metaAdId, metaImageHash, metaCreativeId) locally
- Full targeting: country, age, gender, interests

**OUT of scope:**
- Automated bidding optimization
- Analytics / reporting (reading campaign performance data)
- Custom / lookalike audience creation
- A/B testing
- Scheduling automation
- User-facing UI (API only)

## 3. Data Model

### 3.1 Prisma Models

#### Shared Enum

```prisma
enum MetaCampaignStatus {
  DRAFT
  PAUSED
  ACTIVE
  FAILED
}
```

#### Campaign

```prisma
model Campaign {
  id               String             @id @default(uuid())
  name             String
  objective        String             // "OUTCOME_TRAFFIC" or "OUTCOME_SALES"
  status           MetaCampaignStatus @default(DRAFT)
  metaCampaignId   String?            // Meta's campaign ID after publish
  errorMessage     String?            // Error from Meta API on failure
  createdAt        DateTime           @default(now())
  updatedAt        DateTime           @updatedAt

  adSets           AdSet[]

  @@map("campaigns")
}
```

| Field | Type | Constraints | Default |
|-------|------|-------------|---------|
| `id` | `String` | UUID, PK | `uuid()` |
| `name` | `String` | Required, non-empty | — |
| `objective` | `String` | Must be `OUTCOME_TRAFFIC` or `OUTCOME_SALES` | — |
| `status` | `MetaCampaignStatus` | Enum | `DRAFT` |
| `metaCampaignId` | `String?` | Nullable, set on publish | `null` |
| `errorMessage` | `String?` | Nullable, set on failure | `null` |
| `createdAt` | `DateTime` | Auto | `now()` |
| `updatedAt` | `DateTime` | Auto | `@updatedAt` |

#### AdSet

```prisma
model AdSet {
  id                String             @id @default(uuid())
  campaignId        String
  name              String
  dailyBudget       Int                // In cents (e.g., 1000 = 10.00 EUR)
  targetCountries   String[]           // ISO country codes, e.g. ["NL","DE"]
  targetAgeMin      Int                @default(18)
  targetAgeMax      Int                @default(65)
  targetGenders     Int[]              @default([])  // 1=male, 2=female
  targetInterests   Json               @default("[]") // Array of {id, name}
  optimizationGoal  String             // e.g. "LINK_CLICKS", "OFFSITE_CONVERSIONS"
  billingEvent      String             // e.g. "IMPRESSIONS"
  status            MetaCampaignStatus @default(DRAFT)
  metaAdSetId       String?
  errorMessage      String?
  createdAt         DateTime           @default(now())
  updatedAt         DateTime           @updatedAt

  campaign          Campaign           @relation(fields: [campaignId], references: [id], onDelete: Cascade)
  ads               Ad[]

  @@map("ad_sets")
}
```

| Field | Type | Constraints | Default |
|-------|------|-------------|---------|
| `id` | `String` | UUID, PK | `uuid()` |
| `campaignId` | `String` | FK to Campaign, required | — |
| `name` | `String` | Required, non-empty | — |
| `dailyBudget` | `Int` | Required, positive integer, in cents | — |
| `targetCountries` | `String[]` | Required, at least 1 ISO country code | — |
| `targetAgeMin` | `Int` | 18-65 | `18` |
| `targetAgeMax` | `Int` | 18-65, must be >= targetAgeMin | `65` |
| `targetGenders` | `Int[]` | Values must be 1 or 2 | `[]` |
| `targetInterests` | `Json` | Array of `{id: string, name: string}` | `[]` |
| `optimizationGoal` | `String` | Required | — |
| `billingEvent` | `String` | Required | — |
| `status` | `MetaCampaignStatus` | Enum | `DRAFT` |
| `metaAdSetId` | `String?` | Nullable | `null` |
| `errorMessage` | `String?` | Nullable | `null` |
| `createdAt` | `DateTime` | Auto | `now()` |
| `updatedAt` | `DateTime` | Auto | `@updatedAt` |

**Cascade behavior:** Deleting a Campaign cascades to its AdSets (and their Ads).

## 4. API Endpoints

All endpoints are prefixed with `/api/v1` (set by NestJS global prefix).

### 4.1 POST /api/v1/campaigns — Create Campaign

Creates a local DRAFT campaign. Does NOT call Meta API.

**Request:**
```json
{
  "name": "Summer Sale 2026",
  "objective": "OUTCOME_SALES"
}
```

**Response:** `201 Created`
```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "Summer Sale 2026",
  "objective": "OUTCOME_SALES",
  "status": "DRAFT",
  "metaCampaignId": null,
  "errorMessage": null,
  "createdAt": "2026-03-23T10:00:00.000Z",
  "updatedAt": "2026-03-23T10:00:00.000Z"
}
```

**Errors:**

| Status | Condition | Response |
|--------|-----------|----------|
| 400 | Missing `name` or `objective` | `{ "statusCode": 400, "message": ["name must be longer than or equal to 1 characters"], "error": "Bad Request" }` |
| 400 | Invalid `objective` value | `{ "statusCode": 400, "message": ["objective must be one of the following values: OUTCOME_TRAFFIC, OUTCOME_SALES"], "error": "Bad Request" }` |

### 4.4 PATCH /api/v1/campaigns/:id — Update Campaign

Updates a campaign. If status is `DRAFT`, all fields may be updated. If status is
`PAUSED` or `ACTIVE`, only `name` and `status` may be updated. `FAILED` records
can be freely updated (same as DRAFT) to allow correcting issues before retry.

**Request (DRAFT):**
```json
{
  "name": "Summer Sale 2026 - Updated",
  "objective": "OUTCOME_TRAFFIC"
}
```

**Request (PAUSED/ACTIVE — only name and status):**
```json
{
  "name": "Renamed Campaign",
  "status": "PAUSED"
}
```

**Response:** `200 OK` — Returns the full updated campaign object.

**Errors:**

| Status | Condition | Response |
|--------|-----------|----------|
| 404 | Campaign not found | `{ "statusCode": 404, "message": "Campaign not found", "error": "Not Found" }` |
| 409 | Trying to update locked fields on PAUSED/ACTIVE | `{ "statusCode": 409, "message": "Only name and status can be updated after publishing", "error": "Conflict" }` |

### 4.5 DELETE /api/v1/campaigns/:id — Delete Campaign

Deletes a campaign and all its ad sets and ads (cascade). Only `DRAFT` and `FAILED`
campaigns can be deleted.

**Response:** `204 No Content`

**Errors:**

| Status | Condition | Response |
|--------|-----------|----------|
| 404 | Campaign not found | `{ "statusCode": 404, "message": "Campaign not found", "error": "Not Found" }` |
| 409 | Status is PAUSED or ACTIVE | `{ "statusCode": 409, "message": "Cannot delete a campaign with status ACTIVE. Only DRAFT and FAILED campaigns can be deleted.", "error": "Conflict" }` |

### 4.6 POST /api/v1/campaigns/:id/publish — Publish Campaign to Meta

Pushes the campaign to Meta's Marketing API. Creates a campaign on Meta in
`PAUSED` state, stores the `metaCampaignId`, and updates local status to `PAUSED`.

**Request:** No body required.

**Response:** `200 OK`
```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "Summer Sale 2026",
  "objective": "OUTCOME_SALES",
  "status": "PAUSED",
  "metaCampaignId": "120330000000123456",
  "errorMessage": null,
  "createdAt": "2026-03-23T10:00:00.000Z",
  "updatedAt": "2026-03-23T10:05:00.000Z"
}
```

**Errors:**

| Status | Condition | Response |
|--------|-----------|----------|
| 404 | Campaign not found | `{ "statusCode": 404, "message": "Campaign not found", "error": "Not Found" }` |
| 409 | Already published (metaCampaignId exists and status is not FAILED) | `{ "statusCode": 409, "message": "Campaign is already published", "error": "Conflict" }` |
| 422 | Meta API returned an error | `{ "statusCode": 422, "message": "Meta API error: (#2635) You are not authorized to use this endpoint", "error": "Unprocessable Entity" }` |

**Business logic:**
1. Verify campaign exists and is in `DRAFT` or `FAILED` status
2. Call Meta API: `POST /{ad_account_id}/campaigns` with `name`, `objective`, `status=PAUSED`, `special_ad_categories=[]`
3. On success: store `metaCampaignId`, set status to `PAUSED`, clear `errorMessage`
4. On failure: set status to `FAILED`, store error in `errorMessage`, throw `UnprocessableEntityException`

### 4.18 POST /api/v1/ads/:id/publish — Publish Ad to Meta

The most complex publish flow. Uploads the creative image, creates a Meta creative
object, then creates the ad on Meta. Requires the parent ad set to be published.

**Request:** No body required.

**Response:** `200 OK`
```json
{
  "id": "d4e5f6a7-b8c9-0123-defa-234567890123",
  "adSetId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "name": "Enduro Helmet - Main",
  "creativeId": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "headline": "Premium Enduro Helmets",
  "body": "Ride safe with our top-rated enduro helmets. Free shipping on orders over 50 EUR.",
  "callToAction": "SHOP_NOW",
  "linkUrl": "https://enduro-gear.nl/helmets",
  "status": "PAUSED",
  "metaAdId": "120330000000345678",
  "metaCreativeId": "120330000000567890",
  "metaImageHash": "abc123def456",
  "errorMessage": null,
  "createdAt": "2026-03-23T10:20:00.000Z",
  "updatedAt": "2026-03-23T10:25:00.000Z"
}
```

**Errors:**

| Status | Condition | Response |
|--------|-----------|----------|
| 404 | Ad not found | `{ "statusCode": 404, "message": "Ad not found", "error": "Not Found" }` |
| 409 | Already published (not DRAFT/FAILED) | `{ "statusCode": 409, "message": "Ad is already published", "error": "Conflict" }` |
| 409 | Parent ad set not published | `{ "statusCode": 409, "message": "Parent ad set must be published before publishing an ad", "error": "Conflict" }` |
| 404 | Creative image file not found on disk | `{ "statusCode": 404, "message": "Creative image file not found at uploads/creatives/{creativeId}.png", "error": "Not Found" }` |
| 422 | Meta API error (any step) | `{ "statusCode": 422, "message": "Meta API error: ...", "error": "Unprocessable Entity" }` |

**Business logic (3-step publish):**

1. **Prerequisite check:**
   - Verify ad exists and is `DRAFT` or `FAILED`
   - Load parent ad set; verify `metaAdSetId` is set
   - Verify creative image exists at `uploads/creatives/{creativeId}.png`

2. **Step 1 — Upload image to Meta:**
   - Read file from `uploads/creatives/{creativeId}.png`
   - Call Meta API: `POST /{ad_account_id}/adimages` with the image file
   - Extract `images.{filename}.hash` from response
   - Store `metaImageHash` on Ad record

3. **Step 2 — Create Meta creative:**
   - Call Meta API: `POST /{ad_account_id}/adcreatives` with:
     - `name`: ad name
     - `object_story_spec`: `{ page_id: META_PAGE_ID, link_data: { image_hash: metaImageHash, link: linkUrl, message: body, name: headline, call_to_action: { type: callToAction } } }`
   - Store `metaCreativeId` on Ad record

4. **Step 3 — Create Meta ad:**
   - Call Meta API: `POST /{ad_account_id}/ads` with:
     - `name`: ad name
     - `adset_id`: parent ad set's `metaAdSetId`
     - `creative`: `{ creative_id: metaCreativeId }`
     - `status`: `PAUSED`
   - Store `metaAdId` on Ad record

5. **On success:** Set status to `PAUSED`, clear `errorMessage`
6. **On failure (any step):** Set status to `FAILED`, store error in `errorMessage`.
   If the failure happens at step 2 or 3, previously completed steps (e.g., uploaded
   image) are NOT rolled back — the `metaImageHash`/`metaCreativeId` are kept so a
   retry can skip already-completed steps.

> **Warning: Retry optimization.** On retry (publishing a FAILED ad), check if
> `metaImageHash` already exists and skip the image upload step. Check if
> `metaCreativeId` already exists and skip creative creation. Only re-execute
> steps that haven't completed yet.

## 5. Business Logic

### 5.1 Campaign CRUD

- **Create:** Validate DTO, insert with `status=DRAFT`.
- **List:** Offset pagination `skip = (page - 1) * limit`, `take = limit`. Filter
  by `status` if provided. Return `{ data, total, page, limit }`.
- **Get:** Find by ID, include `adSets` relation (with nested `ads`). Throw
  `NotFoundException` if not found.
- **Update:** Load record, check status. If `DRAFT` or `FAILED`: update any provided
  fields. If `PAUSED`/`ACTIVE`: only allow `name` and `status`; reject other fields
  with `ConflictException`.
- **Delete:** Load record, check status is `DRAFT` or `FAILED`. Otherwise throw
  `ConflictException`. Prisma cascade handles ad sets and ads.

### 5.2 Ad Set CRUD

Same patterns as Campaign CRUD with these additions:
- **Create:** Verify parent campaign exists (`NotFoundException` if not).
- **List:** Additional optional filter by `campaignId`.
- **Get:** Include `ads` relation.
- **Delete:** Prisma cascade handles child ads.

### 5.3 Ad CRUD

Same patterns as Campaign CRUD with these additions:
- **Create:** Verify parent ad set exists AND referenced creative exists.
- **List:** Additional optional filter by `adSetId`.
- **Get:** Include `creative` relation.
- **Update (DRAFT):** If `creativeId` changes, verify the new creative exists.

### 5.4 Publish Flows

All publish flows share this pattern:
1. Load entity, verify it is `DRAFT` or `FAILED`
2. Check prerequisites (parent published, file exists, etc.)
3. Call Meta API
4. On success: store Meta ID, set `status=PAUSED`, clear `errorMessage`
5. On failure: set `status=FAILED`, store error, throw `UnprocessableEntityException`

**Campaign publish:** No prerequisites. Creates Meta campaign with `status=PAUSED`.

**Ad Set publish:** Requires parent campaign to have `metaCampaignId`. Constructs
targeting object from ad set fields.

**Ad publish:** Requires parent ad set to have `metaAdSetId`. Three-step flow
(image upload, creative creation, ad creation). Supports retry optimization by
skipping already-completed steps.

### 5.5 External Service Interactions (Meta Marketing API)

All Meta API calls go through `MetaApiService` — a thin wrapper that:
- Initializes the `facebook-nodejs-business-sdk` with `META_ACCESS_TOKEN`, `META_APP_ID`, `META_APP_SECRET`
- Exposes typed methods for each operation:
  - `createCampaign(adAccountId, params)` — returns `{ id: string }`
  - `createAdSet(adAccountId, params)` — returns `{ id: string }`
  - `uploadImage(adAccountId, imagePath)` — returns `{ hash: string }`
  - `createAdCreative(adAccountId, params)` — returns `{ id: string }`
  - `createAd(adAccountId, params)` — returns `{ id: string }`
- Catches SDK exceptions and re-throws them as a standardized error format
- Uses `ConfigService` to read `META_ACCESS_TOKEN`, `META_APP_ID`, `META_APP_SECRET`,
  `META_AD_ACCOUNT_ID`, `META_PAGE_ID`
- Logs all API calls and responses via NestJS `Logger`

**MetaApiModule** is a global module (or imported by each consumer module) that
exports `MetaApiService`.

> **Warning: SDK initialization.** The `facebook-nodejs-business-sdk` requires
> calling `FacebookAdsApi.init(accessToken)` before making API calls. Do this
> once in the `MetaApiService` constructor or `onModuleInit`. Verify the exact
> initialization pattern against the SDK's current documentation.

## 7. Constraints

1. **No authentication.** Single-tenant service — all endpoints are open (same as
   creative-generation module).
2. **Budget in cents.** `dailyBudget` is always stored and transmitted as an integer
   in cents. No floating-point currency values anywhere in the system.
3. **Meta API creates in PAUSED state.** All entities are created on Meta as `PAUSED`.
   Activating them requires a separate update (PATCH with `status: "ACTIVE"`).
   Updating status on Meta after publish is NOT in scope for v1 — the local status
   changes but Meta status is not synced. Document this as a known limitation.
4. **No rollback on partial publish failure.** If ad publish fails at step 2 or 3,
   artifacts from completed steps remain on Meta. Retry skips completed steps.
5. **Image format.** Only PNG files from `uploads/creatives/{creativeId}.png`. No
   other formats supported.
6. **Interest IDs are opaque.** The system stores and forwards interest IDs to Meta
   without validation. Users must provide valid Meta interest IDs.
7. **Pagination limits.** Default 20, max 100 per page. Page numbers start at 1.
8. **UUID validation.** All `:id` path params use `ParseUUIDPipe` (same as creatives module).
9. **Global prefix.** All routes use the NestJS global prefix `api/v1` (configured in `main.ts`).
10. **No status sync from Meta.** Local status reflects what was pushed, but does not
    poll Meta for status changes. If a campaign is paused/activated directly in Meta
    Ads Manager, the local DB will be out of sync. Out of scope for v1.

> **Known limitation (v1):** Updating `status` to `ACTIVE` via PATCH only changes
> the local DB status. It does NOT call Meta API to activate the campaign/ad set/ad.
> A future version should add Meta status sync on update.
